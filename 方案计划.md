# AI 求职智能匹配智能体 — 实现计划

## 1. 目标与交付物（对应题目要求）
- **痛点1（找岗难）**：从海量岗位中快速匹配契合学生背景/能力/兴趣的岗位。
- **痛点2（简历不知好坏）**：评估简历与目标岗位匹配度，给出可提升初筛命中率的优化建议。
- **交付物①**：可运行完整 Demo（Web 网页），**部署公网**并提供访问链接。
- **交付物②**：1000 字以内方案说明（DOC/PDF），含问题诊断、方案设计、AI 选型理由、关键配置、迭代记录、效果评估。

## 2. 已确认的关键决策
- Demo 形式：**Web 网页**（Next.js 全栈）。
- 数据源：**公开真实数据集**（Hugging Face / Kaggle），真实、合规、演示稳定。
  - 候选：`netsol/resume-score-details`（简历+JD+评分）、SkillSkape（合成 JD）、Kaggle job postings。
- AI 能力：**Claude API（Anthropic，用户提供 Key）**，做简历解析、语义匹配解释、简历诊断与优化改写。
  - 注意：Anthropic 不提供 embedding 接口。**召回阶段用本地轻量开源 embedding**（Node 端 `@xenova/transformers` 跑小模型），分析/生成全部用 Claude。纯 Claude 生态、零额外 Key。
- 技术栈：**Next.js 全栈 + Vercel 部署**。

## 3. 系统架构
```
用户上传简历(文本/粘贴)
        │
        ▼
[简历解析] ── LLM 抽取结构化字段(技能/经历/教育/求职意向)
        │
        ▼
[岗位匹配] ── Embedding 语义检索 + 多维打分(技能/经历/兴趣) → Top-N 岗位
        │
        ▼
[匹配解释] ── LLM 解释为何匹配 + 匹配度评分
        │
        ▼
[简历诊断] ── 针对选定岗位:命中/缺失关键词、初筛命中率评估、优化建议、改写示例
```

### 数据层
- 离线脚本下载并清洗公开数据集 → 统一岗位 schema（标题/公司/技能/JD/薪资/地点/经验要求）。
- 预计算岗位 embedding，存为本地 JSON / 轻量向量文件，随项目打包（部署时无需外部 DB）。

### 匹配算法（两阶段，业界成熟做法）
1. **召回**：用 embedding 余弦相似度从岗位库取 Top-K。
2. **精排+解释**：组合分 = 语义相似度 + 技能标签重合 + 求职意向匹配；LLM 生成匹配理由。

### 简历优化（提升初筛命中率）
- 对选定岗位，对比简历与 JD：列出命中/缺失关键词与技能。
- 给出初筛命中率估计 + 具体优化项 + 针对该 JD 的改写示例（STAR 化、量化成果、关键词补齐）。

## 4. 技术栈
- 前端/后端：Next.js (App Router) + TypeScript + Tailwind。
- AI：**Claude（Anthropic）**做 Chat 类任务（解析/匹配解释/诊断/改写）；**召回用本地 `@xenova/transformers` embedding**。Key 走环境变量，不硬编码。
- 数据：构建期生成的静态岗位库 + 预计算向量（JSON）。
- 部署：Vercel（公网链接），环境变量配置 API Key。

## 5. 实施步骤
1. 初始化 Next.js 项目（TS + Tailwind）。
2. 数据管线：写脚本拉取+清洗公开数据集，生成岗位库与 embedding，落地到 `data/`。
3. 后端 API：
   - `/api/parse-resume` 简历解析
   - `/api/match` 岗位匹配（召回+精排+解释）
   - `/api/optimize` 简历诊断与优化
4. 前端页面：简历输入 → 匹配岗位列表（含匹配度/理由）→ 选岗位看诊断与优化建议。
5. LLM 适配层：抽象 chat/embedding 接口，便于切换模型；Key 用 env。
6. 本地联调验证全流程。
7. 部署 Vercel，配置环境变量，产出公网链接。
8. 写方案说明文档（≤1000 字，含迭代记录与效果评估），导出 DOC/PDF。

## 6. 安全与合规
- API Key 仅走环境变量，不写入代码/仓库。
- LLM 调用在服务端，不在前端暴露 Key。
- 数据用公开数据集并标注出处，规避爬虫合规风险。
- 部署为公网服务：Demo 不涉及真实用户隐私数据，简历由用户即时输入、不持久化存储。
